## Práctica evaluable de Android
**Grupo**

Hilaria Romero Bouyahia , Paula Ríos Sánchez

**Tema** 
> Crearemos una aplicación que junte control de inventario informado y lista de la compra con el objetivo de crear un flujo de trabajo sencillo y cómodo para facilitarle al usuario el proceso de controlar qué posee, su opinión de ellos, apuntar los productos que faltan para su próxima compra y repetir.

**Lenguaje**

Java

### Requisitos:
- Android 14.0 (API 34).
- Dos actividades:
    - Inventario: lista con los productos en posesión, aumentar o disminuir cantidades y añadir el producto a la lista de la compra
    - Lista de la compra: Lista de objetos para comprar, eliminar de la lista y se añadir al inventario
- Notificaciones y ventanas de información de errores para los usuarios
    - Toast y Snackbar: Operaciones habituales con los objetos
    - Dialogs: Error al comprar cantidades no válidas o cantidad negativa de productos en el inventario
- Conexión con una base de datos MongoDB
- Soporte multi-idioma en español e inglés
- Abstracción, diseño y eficiencia de los componentes del sistema.
